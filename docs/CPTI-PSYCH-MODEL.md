# CPTI 心理学引擎技术文档

CPTI(关系塑 · Couple Personality Test Indicator)是一套双人感情测试系统。
它不发明心理学,而是把 5 个已验证的经典模型缝合成一张坐标网,把答题者坐标化为 **4 主轴 + 2 副轴**,再投射为 **6 大 CPTI 类型**,最后在 **6×6 = 36** 种配对里给出关系诊断。

本文不是论文,是工程文档 —— 说明"每一题挂在哪条轴上、每条轴怎么合成成类型、每只动物为什么落在这一格"。

---

## 1. 五个底层模型

### 1.1 Bowlby / Ainsworth 依恋理论(主轴)

**来源**:Bowlby 的婴儿依恋研究(1969) + Ainsworth 的陌生情境实验(1978),后来 Bartholomew & Horowitz (1991) 把成人依恋模型化为 **焦虑 × 回避** 二维坐标。

**关键概念**:
- **焦虑(Anxiety)**:担心被抛弃、过度监测关系温度,高焦虑会不停追问"你是不是不爱我了"。
- **回避(Avoidance)**:抗拒亲密、自我披露低,高回避会在靠得太近时本能撤退。
- 两维交叉出 4 型:**安全型**(低焦低避)、**焦虑型**(高焦低避)、**回避型**(低焦高避)、**紊乱型**(高焦高避)。

**在 CPTI 怎么落地**:焦虑和回避直接就是 CPTI 的两条主轴(见 §2),构成整个测试的底盘。6 大类型里 PURS 继承焦虑型,AVOI 继承回避型,ANCH 继承安全型,CHAO 直接等于紊乱型。

### 1.2 Sternberg 爱情三角理论(主轴 + 副轴)

**来源**:Robert Sternberg (1986) 提出爱情由三种成分构成,不同权重组合出 7 种爱情形态。

**关键概念**:
- **激情(Passion)**:生理唤醒、心动、占有欲 —— 燃点高、冷却也快。
- **亲密(Intimacy)**:情感连接、自我披露、相互理解 —— 慢燃但持久。
- **承诺(Commitment)**:决定保持这段关系、愿意长期投入 —— 理性层的选择。

**在 CPTI 怎么落地**:激情和承诺作为第三、第四条主轴与依恋二维垂直,用来区分 **"高焦却不激情"(如 kitsune)** 与 **"高焦且激情拉满"(如 husky)**;亲密度则作为副轴,专门校准 WEAV 的编织浓度。

### 1.3 Gottman 四骑士理论(副轴 · 冲突风格)

**来源**:John Gottman 通过 40 年"爱情实验室"数据统计出,能 **94% 预测离婚的 4 种互动模式**(Four Horsemen):批评、蔑视、防御、筑墙。

**关键概念**:出现四骑士不必然离婚,关键在于 **修复尝试(repair attempts)** 的频率 —— 安全的伴侣吵架多,但每次都能拉回来;紊乱的伴侣越吵越升级。

**在 CPTI 怎么落地**:CPTI 不直接测四骑士的出现频率(太深),但用它的"修复倾向"作为 36 配对算法里的 **冲突走向系数** —— 例如 CHAO × CHAO 组合会被标记为"需要专业修复模板",ANCH × 任何型则天然有修复优势。

### 1.4 Chapman 五种爱的语言(副轴 · 表达层)

**来源**:Gary Chapman (1992)《The Five Love Languages》。观察到人感受爱的主通道各不相同。

**关键概念**:肯定言辞、精心时刻、赠送礼物、服务行动、身体接触 —— 人 **给出** 的语言和 **想收** 的语言常常错位。

**在 CPTI 怎么落地**:在测试尾部用 6 题测表达偏好,结果不进入主轴,但用于 **WEAV 编织者的副类型判定** 以及 36 配对里的"语言匹配度"补充分数。错位本身不是病,关键看能不能翻译。

### 1.5 Rusbult 投资模型(副轴 · 稳定性)

**来源**:Caryl Rusbult (1980) 扩展了社会交换理论,提出关系稳定度 = **满意度 + 投资量 − 替代选项吸引力**。

**关键概念**:
- **满意度(Satisfaction)**:当前关系的快乐水平。
- **投资(Investment)**:为这段关系付出的时间、精力、共同资源 —— 越多越难抽身。
- **替代项(Alternatives)**:外部机会的吸引力。

**在 CPTI 怎么落地**:作为"黏性"变量,专门拉开 KIND 和 ANCH 的区别 —— 两者都可能"当前感觉好",但 KIND 的投资低、替代项高,ANCH 正好相反。

## 2. 主副轴 → 题目映射

CPTI 共 24 题,每题挂在 **1 条主轴 + (可选)1 条副轴**。这里给出轴→题的索引(题 ID 1-24,供出题工程师对齐)。

| 轴(维度) | 模型来源 | 题号(主挂) | 题号(辅挂) |
|---|---|---|---|
| 焦虑(Anxiety) | Bowlby/Ainsworth | Q1, Q5, Q9, Q13 | Q17, Q21 |
| 回避(Avoidance) | Bowlby/Ainsworth | Q2, Q6, Q10, Q14 | Q18, Q22 |
| 激情(Passion) | Sternberg | Q3, Q7, Q11, Q15 | Q19 |
| 承诺(Commitment) | Sternberg / Rusbult | Q4, Q8, Q12, Q16 | Q20 |
| 亲密(Intimacy · 副轴) | Sternberg | Q17, Q18, Q19, Q20 | — |
| 表达(Expression · 副轴) | Chapman | Q21, Q22, Q23, Q24 | — |

**打分规则**:每题 5 档 Likert(-2 ~ +2),主挂计 1.0×,辅挂计 0.5×。主轴结果归一化到 [-4, +4] 区间,副轴归一化到 [-2, +2]。

```
焦虑轴示意:
  -4 ─────── 0 ─────── +4
  安全      中立      高焦(追问型)
  
回避轴示意:
  -4 ─────── 0 ─────── +4
  亲近      中立      高避(筑墙型)
```

## 3. 6 大类型派生公式

定义四条主轴坐标 `(Ax, Av, Ps, Cm)` = (焦虑, 回避, 激情, 承诺),亲密副轴为 `In`,表达副轴为 `Ex`。

```
┌──────────────────────────────────────────────────────┐
│  派生伪代码(优先级从上到下,首个命中即为 primary)    │
├──────────────────────────────────────────────────────┤
│  IF  Ax > 1.5  AND  Av > 1.5                          │
│      → CHAO  (风暴眼 · 紊乱型)                        │
│  ELIF  Av > 1.5  AND  Ax <= 1.5                       │
│      → AVOI  (回避者)                                 │
│  ELIF  Ax > 1.5  AND  Ps > 1.0                        │
│      → PURS  (追求者)                                 │
│  ELIF  Ps > 1.5  AND  Cm < 0                          │
│      → KIND  (点火者)                                 │
│  ELIF  In > 1.0  AND  Ex > 0.5                        │
│      → WEAV  (编织者)                                 │
│  ELSE                                                 │
│      → ANCH  (定锚人 · 安全型默认锚)                  │
└──────────────────────────────────────────────────────┘
```

| Code | 中文 | 核心坐标特征 | 一句话 |
|---|---|---|---|
| **PURS** | 追求者 | Ax↑ Ps↑ | 焦虑燃料 + 激情外显,一头扎进去不回头 |
| **AVOI** | 回避者 | Av↑ (Ax 低) | 低自我披露 + 低激情,你越近我越退 |
| **ANCH** | 定锚人 | Ax↓ Av↓ Cm↑ | 低焦低避 + 高承诺,爱不刺激但稳 |
| **CHAO** | 风暴眼 | Ax↑ Av↑ | 既要又要,自己也累 |
| **KIND** | 点火者 | Ps↑ Cm↓ | 激情高但投资低,第一晚最热 |
| **WEAV** | 编织者 | In↑ Ex↑ | 高亲密+高情感语言,把日子织成网 |

**Secondary 派生**:主类型确定后,去掉最高权重条件后重跑一次公式,得到 secondary。这样每人是"PURS·KIND"或"AVOI·ANCH"这样的 **主副复合型**,比单一类型更可读。

## 5. 36 配对算法

6 大类型两两组合,产生 **C(6,1)² = 36** 种配对(含自配)。算法分三层:

### 5.1 Layer 1 · 兼容性指数(0-100)

对每一对 `(T1, T2)`,基于依恋理论里的"安全—不安全"组合基线打分:

```
           PURS  AVOI  ANCH  CHAO  KIND  WEAV
  PURS      55    35    80    30    50    70
  AVOI      35    40    75    25    45    50
  ANCH      80    75    90    70    75    85
  CHAO      30    25    70    20    35    55
  KIND      50    45    75    35    55    65
  WEAV      70    50    85    55    65    78
```

规则:
- ANCH 和任何型配对基线都高(安全型是缓冲器)。
- CHAO × CHAO 和 CHAO × AVOI 是风险组合。
- 同型配对除 ANCH·ANCH 外都偏中等,因为放大同向偏差。

### 5.2 Layer 2 · 三个修正系数

在基线分数上叠加:

| 系数 | 公式 | 作用 |
|---|---|---|
| **激情热度** | `clamp(Ps_1 + Ps_2, -2, +2) × 3` | 两人激情同向加分,互相抵消扣分 |
| **承诺厚度** | `min(Cm_1, Cm_2) × 4` | 短板决定承诺下限 |
| **语言匹配** | Jaccard(Ex_1, Ex_2) × 10 | 表达主通道重合度 |

### 5.3 Layer 3 · 诊断文本

最终分段:

- **90-100** 天生一对 —— 依恋兼容 + 激情承诺都够
- **75-89** 有戏可走 —— 主轴对齐但需一两处翻译
- **60-74** 能谈但要功课 —— 有一个方向差异需修复模板
- **45-59** 高风险高回报 —— 化学反应强但冲突模式需注意
- **< 45** 建议重考或接受差异 —— 四骑士模式易升级

每种配对还会生成 3 条 **关系剧本**(第一次吵架、第一次冷战、第一次复合)作为可读性内容,供前端展示。
